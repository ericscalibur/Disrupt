# VPS Deployment Guide

This guide covers migrating Disrupt Portal from local to a production VPS with HTTPS.

## Prerequisites

- A VPS running Ubuntu 22.04+ (DigitalOcean, Hetzner, Vultr, etc.)
- A domain name pointed at the VPS IP (A record, e.g. `pay.yourcompany.com`)
- SSH access to the VPS

---

## 1. Install Dependencies on the VPS

```bash
# Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Nginx
sudo apt install -y nginx

# PM2 (keeps the app running after reboots)
sudo npm install -g pm2

# Certbot (Let's Encrypt)
sudo apt install -y certbot python3-certbot-nginx
```

---

## 2. Deploy the App

```bash
# Clone the repo
git clone https://github.com/ericscalibur/Disrupt.git /home/disrupt
cd /home/disrupt

# Install dependencies
npm install

# Run first-time setup (creates Admin user, JWT secrets, DB)
npm run setup
```

Set up the `.env` file:

```bash
cp .env.example .env   # if it exists, otherwise create it manually
nano .env
```

`.env` should contain:

```
NODE_ENV=production
PORT=3000
ACCESS_TOKEN_SECRET=<generated by setup>
REFRESH_TOKEN_SECRET=<generated by setup>
BLINK_API_KEY=<your Blink key>
TAX_LIGHTNING_ADDRESS=<your tax lightning address>
EMAIL_USER=<gmail address>
EMAIL_PASS=<gmail app password>
```

---

## 3. Start the App with PM2

```bash
cd /home/disrupt
pm2 start disrupt-portal/server.js --name disrupt
pm2 save
pm2 startup   # follow the printed command to enable auto-start on reboot
```

Verify it's running:
```bash
pm2 status
curl http://localhost:3000/api/btc-usd-rate
```

---

## 4. Configure Nginx

Create the site config (replace `pay.yourcompany.com` with your domain):

```bash
sudo nano /etc/nginx/sites-available/disrupt
```

Paste:

```nginx
server {
    listen 80;
    server_name pay.yourcompany.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

Enable it:

```bash
sudo ln -s /etc/nginx/sites-available/disrupt /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

---

## 5. Enable HTTPS with Let's Encrypt

```bash
sudo certbot --nginx -d pay.yourcompany.com
```

Certbot will automatically update the Nginx config to handle HTTPS and redirect HTTP → HTTPS. Auto-renewal is set up automatically.

Verify renewal works:
```bash
sudo certbot renew --dry-run
```

---

## 6. Firewall — Block Direct Port 3000 Access

Only Nginx should be able to reach the app. Block port 3000 from the public internet:

```bash
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'   # ports 80 and 443
sudo ufw deny 3000
sudo ufw enable
sudo ufw status
```

---

## 7. Verify Everything

- `https://pay.yourcompany.com` — loads the app over HTTPS
- `http://pay.yourcompany.com` — redirects to HTTPS automatically
- `http://pay.yourcompany.com:3000` — connection refused (blocked by firewall)

---

```bash
cd /home/disrupt

```
